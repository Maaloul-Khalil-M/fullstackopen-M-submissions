# Deploy our app to the internet

## start with the back

render.com free tier allow us to host our simple API on the web
on it we create a new web service and connect render to our repo

apply: const PORT = process.env.PORT || 3001

## prove it

## limit of render free tier

- If API isn't visited for 15 minutes, Render puts your server to "sleep" to save resources.
- The first request will hang for 30–60 seconds
- Hardware: 0.1 CPU & 512 MB RAM
- No Persistent Disk
- No SSH Access and no instant logs (define time range)
- Scaling: Single Instance Only
- No One-Off Jobs & Background Workers

## adding the minified front

in front we need minified by npm run build it create a /dist

we move it to back: move "FRONT\dist" "BACK"

and we serve those static files by: app.use(express.static('dist'))

and now with everything in one folder meaning have the same adress
we can declate baseURL as relative url
const baseUrl = "/api/persons"

we can now lunch both the front and back from the backurl port 3001

# 🧠 The core idea (no confusion version)

👉 You do NOT push `dist/` because:

> Git stores **source code**, not generated output.

`dist/` is:

- generated by `npm run build`
- always reproducible
- not written by you

So Git stores the recipe, not the cooked food.

---

# 🚀 Your automation script (correct + simple)

Put this in **backend `package.json`**:

```json
"scripts": {
  "build:ui": "cd ../phonebook && npm run build && xcopy dist ..\\PhonebookBackend\\dist /E /I /Y",
  "deploy:full": "npm run build:ui && git add . && git commit -m \"ui build\" && git push"
}
```
